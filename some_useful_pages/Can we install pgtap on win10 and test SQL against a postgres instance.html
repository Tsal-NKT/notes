    Hi @theory I was able to install on Windows 10 with PG11 here are my procedures:

    1. Use https://strawberryperl.com/
    2. git clone https://github.com/theory/pgtap.git
    3. Run portable_shell.bat from strawaberryperl an did the following steps:
    cd pgtapFolder
copy sql/pgtap.sql.in sql\pgtap.sql
perl.exe -pi.bak -e "s/TAPSCHEMA/tap/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/__OS__/win32/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/__VERSION__/0.24/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/^-- ## //g" sql\pgtap.sql
copy sql\pgtap.sql "%ProgramFiles%\PostgreSQL\11\share\extensions"
copy contribut\pgtap.spec  "%ProgramFiles%\PostgreSQL\11\contribut"
copy pgtap.control "%ProgramFiles%\PostgreSQL\11\share\extensions"
ren "%ProgramFiles%\PostgreSQL\10\share\extensions\pgtap.sql" "%ProgramFiles%\PostgreSQL\11\share\extensions\pgtap--1.1.0.sql"

    After this, I was able to install using PgAdmin4.

    I think this can be polished but for now I was able to run it.

    ericpyle commented Nov 3, 2021 •
    edited

    My steps were a little different:

    1. Go to https://strawberryperl.com/releases.html and click on the Portable 64bit download link then extract to {strawberryperl} folder

    2. git clone https://github.com/theory/pgtap.git into {pgtapFolder}

    3. Run cmd.exe as Admininstrator (this is so you can copy into ProgramFiles)

    4. cd to the extracted portable {strawberryperl} (see step 1) folder and run "portableshell.bat"

    5. Then, do the following steps:

    cd {pgtapFolder}
copy sql\pgtap.sql.in sql\pgtap.sql
perl.exe -pi.bak -e "s/TAPSCHEMA/tap/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/__OS__/win32/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/__VERSION__/0.24/g" sql\pgtap.sql
perl.exe -pi.bak -e "s/^-- ## //g" sql\pgtap.sql
copy sql\pgtap.sql "%ProgramFiles%\PostgreSQL\12\share\extension"
copy contrib\pgtap.spec  "%ProgramFiles%\PostgreSQL\12\contrib"
copy pgtap.control "%ProgramFiles%\PostgreSQL\12\share\extension"
cd "%ProgramFiles%\PostgreSQL\12\share\extension\"
ren "pgtap.sql" "pgtap--1.2.0.sql"

    If after trying to CREATE the extension you get an error like "error: extension "pgtap" has no installation script nor update path for version "{version}", then change the last step above to ren "pgtap.sql" "pgtap--{version}.sql"
